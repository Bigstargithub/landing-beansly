# 커피 원두 랜딩페이지 와이어프레임

## 페이지 개요
- **목표**: 커피를 즐기는 사람들이 쉽게 원두를 찾고 구독할 수 있도록 유도
- **핵심 가치**: 편리함, 맞춤 추천, 품질 보증
- **주요 CTA**: 이메일 수집 (얼리버드 알림 신청)

---

## 섹션 구조

### 1. Hero 섹션
**위치**: 첫 화면 (Above the fold)

**구성 요소**:
- 헤드라인 (H1): 문제 공감 + 솔루션 제시
- 서브헤드라인 (H2): 가치 제안 명확화
- CTA 버튼: "지금 시작하기" → CTA 섹션으로 스크롤
- 배경 이미지: 따뜻한 커피 이미지 또는 그라데이션

**레이아웃**:
```
┌─────────────────────────────────────┐
│                                     │
│         [로고]                       │
│                                     │
│    매일 마시는 커피,                  │
│    이제 더 쉽게 찾아보세요            │
│                                     │
│    취향에 딱 맞는 원두를              │
│    집 앞까지 배송해드립니다            │
│                                     │
│      [지금 시작하기 버튼]             │
│                                     │
└─────────────────────────────────────┘
```

**의도**: 
- 사용자가 느끼는 불편함("원두 찾기 어려움")을 즉시 공감하고
- 해결책("쉽게 찾고, 배송까지")을 명확히 제시하여
- 첫 3초 안에 이탈을 막습니다.

---

### 2. Features/Value 섹션
**위치**: Hero 바로 아래

**구성 요소**:
3가지 핵심 가치를 카드 형태로 배치

1. **맞춤 추천**
   - 아이콘: 🎯 또는 체크 아이콘
   - 헤드라인: "취향에 딱 맞는 원두"
   - 설명: "간단한 질문으로 나에게 맞는 원두를 추천받으세요"

2. **편리한 배송**
   - 아이콘: 🚚 또는 배송 아이콘
   - 헤드라인: "집 앞까지 신선하게"
   - 설명: "로스팅 직후 신선한 원두를 집 앞까지 배송해드립니다"

3. **품질 보증**
   - 아이콘: ⭐ 또는 인증 아이콘
   - 헤드라인: "엄선된 원두만"
   - 설명: "전문가가 직접 테스트한 고품질 원두만 제공합니다"

**레이아웃**:
```
┌───────────────────────────────────────┐
│                                       │
│     이런 점이 좋아요                    │
│                                       │
│  ┌─────┐  ┌─────┐  ┌─────┐          │
│  │ 🎯  │  │ 🚚  │  │ ⭐  │          │
│  │맞춤 │  │편리한│  │품질 │          │
│  │추천 │  │배송 │  │보증 │          │
│  └─────┘  └─────┘  └─────┘          │
│                                       │
└───────────────────────────────────────┘
```

**의도**:
- 사용자가 얻는 구체적 이점을 3가지로 압축하여
- 한눈에 파악할 수 있도록 단순화했습니다.
- 기술이 아닌 혜택 중심으로 설명합니다.

---

### 3. Proof 섹션
**위치**: Features 아래

**구성 요소**:
- 사용자 후기 2-3개 (간결한 한 문장)
- 또는 신뢰 지표 (예: "1,000명 이상이 기다리고 있어요")

**레이아웃**:
```
┌───────────────────────────────────────┐
│                                       │
│     이미 많은 분들이 기다리고 있어요     │
│                                       │
│  "정말 취향에 딱 맞는 원두를 받았어요"  │
│  - 김○○                               │
│                                       │
│  "더 이상 원두 찾느라 헤매지 않아요"   │
│  - 이○○                               │
│                                       │
└───────────────────────────────────────┘
```

**의도**:
- 실제 사용자의 목소리로 신뢰를 구축하고
- 아직 런칭 전이라면 대기자 수로 사회적 증거를 제시합니다.

---

### 4. CTA 섹션
**위치**: 페이지 하단 (Proof 아래)

**구성 요소**:
- 헤드라인: 행동 유도 메시지
- 이메일 입력 폼
- 제출 버튼
- 개인정보 처리 동의 문구 (선택)

**레이아웃**:
```
┌───────────────────────────────────────┐
│                                       │
│     지금 신청하고 혜택 받으세요          │
│                                       │
│  런칭 소식과 특별 할인을 가장 먼저      │
│  받아보세요                            │
│                                       │
│  ┌─────────────────────────────┐     │
│  │ 이메일을 입력해주세요         │     │
│  └─────────────────────────────┘     │
│                                       │
│       [알림 신청하기 버튼]             │
│                                       │
│  * 이메일은 알림 목적으로만 사용됩니다   │
│                                       │
└───────────────────────────────────────┘
```

**Google Sheets 연동**:
- 이메일 입력 시 자동으로 Google Sheets에 저장
- 저장 항목: 이메일, 신청 일시, (선택) 관심 원두 타입

**의도**:
- 페이지 전체의 최종 목표인 이메일 수집에 집중하고
- 간단한 입력만으로 완료할 수 있도록 마찰을 최소화했습니다.
- 개인정보 사용에 대한 투명성을 확보합니다.

## 디자인 원칙

### 1. 깔끔하고 집중도 높은 구조
- 섹션 간 충분한 여백 (최소 80px)
- 한 화면에 하나의 메시지만
- 스크롤 없이 Hero 섹션 완전 노출

### 2. 반응형 디자인
- 모바일: 단일 컬럼, 터치 친화적 버튼 크기
- 태블릿: Features를 2열로 배치
- 데스크톱: Features를 3열로 배치

### 3. 색상 및 타이포그래피
- 주 색상: 따뜻한 브라운 계열 (커피 연상)
- 보조 색상: 크림/베이지 (부드러운 느낌)
- 헤드라인: 크고 굵게 (48-64px)
- 본문: 읽기 편한 크기 (16-18px)

### 4. 접근성
- 충분한 색상 대비 (WCAG AA 이상)
- 키보드 네비게이션 지원
- 스크린리더 호환 (semantic HTML)

---

## 페이지 플로우

1. 방문자 도착 → Hero 섹션에서 문제 공감
2. 스크롤 → Features에서 해결책 이해
3. 스크롤 → Proof에서 신뢰 구축
4. 스크롤 → CTA에서 이메일 입력
5. 제출 → Google Sheets 저장 → 감사 메시지 표시

**예상 체류 시간**: 30-60초
**목표 전환율**: 20-30% (이메일 신청률)
