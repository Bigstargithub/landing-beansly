# AGENTS

본 프로젝트에서 사용하는 AI 에이전트들의 역할, 입력/출력 형식, 완료 기준을 정의한다.

## 1) Planner (기획 에이전트)
- 목적: 요구사항을 랜딩페이지 구조/문서로 변환
- 입력: 비즈니스 목표, 타깃, 핵심 메시지(한 문단)
- 출력: IA(섹션 목록), 각 섹션의 목적/핵심 카피, 와이어프레임 문서(Markdown)
- 완료 기준: 필수 섹션 포함, 톤 앤 매너 준수, CTA 명확

## 2) UI Builder (구현 에이전트)
- 목적: Next.js + Tailwind 구성요소 생성
- 입력: Planner 산출물(IA/카피), 디자인 제약(cursorrules)
- 출력: 컴포넌트 코드, 경로(app/), 샘플 에셋 배치, 미리보기 방법
- 완료 기준: 빌드 성공, a11y/SEO 체크리스트 통과

## 3) Copy Refiner (문구 다듬기)
- 목적: 섹션 카피의 명확성/전환율 개선
- 입력: 섹션 원문, 페르소나, 혜택/문제/증거 요소
- 출력: 3가지 대안 카피 + 추천안 1개(선정 이유 포함)
- 완료 기준: 브랜드 톤 준수, i18n 키 사용

## 4) QA Reviewer (품질 점검)
- 목적: 링크/레이아웃/반응형/문구/맞춤법 점검
- 입력: 배포 프리뷰 URL 또는 로컬 캡처
- 출력: 이슈 리스트(심각도/재현/수정 제안), 체크리스트 결과
- 완료 기준: Blocker 0, Major 0 상태

## 5) Release Manager (배포 관리)
- 목적: 브랜치 전략에 따른 배포 수행
- 입력: PR/커밋 메시지, 변경 로그
- 출력: 프리뷰/프로덕션 배포 링크, 간단 릴리즈 노트
- 완료 기준: main 병합 시 자동 배포 성공, 롤백 절차 문서화

## 공통 원칙
- 모든 에이전트는 `.cursorrules`를 우선 준수한다.
- 산출물은 한국어 설명을 포함하고, 비개발자도 이해 가능해야 한다.
- 설명은 “왜 이렇게 했는가(의도)”를 3~5줄로 남긴다.
